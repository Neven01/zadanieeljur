1. ОСНОВНЫЕ ПОНЯТИЯ ООП

Класс – это абстрактный шаблон или "чертеж" для создания объектов-полиномов.
В нашем задании: класс Polynomial определяет структуру (односвязный список) 
и поведение (операции с полиномами) всех создаваемых полиномов.

Объект (Экземпляр) – это конкретный полином, созданный на основе класса.
Например: 
  poly1 = Polynomial([1, 2, 3])  # объект полинома 3x² + 2x + 1
  poly2 = Polynomial([0, 1])     # объект полинома x

Атрибут (Поле) – переменная, хранящая данные объекта.
В классе Polynomial:
  - self.head    # указатель на начало списка узлов
  - self._size   # количество членов полинома
В классе _Node:
  - self.coefficient  # коэффициент члена полинома
  - self.exponent     # степень члена полинома
  - self.next         # указатель на следующий узел

Метод – функция, определенная внутри класса, описывающая поведение объектов.
В нашем задании:
  - __add__()       # сложение полиномов
  - __mul__()       # умножение полиномов  
  - differentiate() # дифференцирование
  - clone()         # создание копии (паттерн Прототип)

2. ВНУТРЕННИЕ (ВЛОЖЕННЫЕ) КЛАССЫ

Класс _Node является внутренним классом класса Polynomial:
- Определен внутри области видимости Polynomial
- Используется только для реализации Polynomial
- Инкапсулирует логику хранения отдельных членов полинома

Преимущества внутреннего класса:
- Логическая группировка: узел не существует без полинома
- Инкапсуляция: детали реализации скрыты от внешнего кода
- Контроль доступа: внешний код не может создавать узлы напрямую

3. ТИПЫ ОТНОШЕНИЙ МЕЖДУ КЛАССАМИ

3.1 КОМПОЗИЦИЯ ("содержит", строгое часть-целое)
Отношение: Polynomial ○───> _Node

Характеристики:
- Объект-часть (_Node) не может существовать без объекта-целого (Polynomial)
- Жизненные циклы связаны: при удалении полинома удаляются все его узлы
- Управление созданием: узлы создаются только методами полинома

Пример в коде:
class Polynomial:
    class _Node:          # узел определен внутри полинома
        pass
    
    def add_term(self, coefficient, exponent):
        new_node = self._Node(coefficient, exponent)  # полином создает узлы
        # узел не имеет смысла без полинома

3.2 АССОЦИАЦИЯ ("использует")
Отношение: Polynomial ───> Polynomial

Характеристики:
- Объекты знают друг о друге и взаимодействуют
- Жизненные циклы независимы
- Слабая связь: объекты могут существовать по отдельности

Пример в коде:
def __add__(self, other):  # other - другой полином
    result = Polynomial()
    # self использует other для операции сложения
    # но они существуют независимо друг от друга

3.3 ПАТТЕРН ПРОТОТИП ("клонирует")
Отношение: Polynomial ───> Polynomial (создание копий)

Характеристики:
- Объект может создавать свои копии
- Копия наследует состояние оригинала
- Полезно для создания объектов с сложной инициализацией

Пример в коде:
def clone(self):
    return copy.deepcopy(self)  # создание полной копии

# Использование:
original = Polynomial([1, 2, 3])
copy_poly = original.clone()  # копия с тем же состоянием

4. РЕАЛИЗАЦИЯ ОПЕРАЦИЙ С ПОЛИНОМАМИ

4.1 Сложение (__add__)
- Создает новый полином-результат
- Последовательно добавляет члены из обоих полиномов
- Объединяет члены с одинаковыми степенями

4.2 Умножение (__mul__)  
- Перемножает каждый член первого полинома с каждым членом второго
- Складывает члены с одинаковыми степенями
- Использует вложенные циклы для обхода обоих полиномов

4.3 Дифференцирование (differentiate)
- Применяет формулу: d/dx(a*x^n) = (a*n)*x^(n-1)
- Уменьшает степень каждого члена на 1
- Умножает коэффициент на исходную степень

4.4 Сравнение (__eq__)
- Сравнивает полиномы на идентичность
- Проверяет совпадение всех членов (коэффициентов и степеней)
- Учитывает порядок членов в списке

5. ПАТТЕРНЫ ПРОЕКТИРОВАНИЯ

5.1 Паттерн ПРОТОТИП (Prototype)
Назначение: создание новых объектов путем копирования существующих

Реализация в задании:
- Метод clone() создает глубокую копию полинома
- Использует copy.deepcopy() для рекурсивного копирования всех узлов
- Сохраняет структуру и состояние оригинального полинома

Преимущества:
- Избегает сложной повторной инициализации
- Сохраняет состояние объекта
- Позволяет создавать модифицированные копии

6. СТРУКТУРА ДАННЫХ: ОДНОСВЯЗНЫЙ СПИСОК

Организация полинома:
- Члены хранятся в узлах списка
- Список отсортирован по убыванию степеней
- Каждый узел содержит: коэффициент, степень, ссылку на следующий узел

Преимущества для полиномов:
- Эффективное добавление/удаление членов
- Естественное представление разреженных полиномов
- Гибкость при изменении структуры

7. ТЕСТИРОВАНИЕ

Подходы к тестированию:
- Модульное тестирование отдельных операций
- Проверка граничных случаев (нулевые полиномы, одинаковые полиномы)
- Тестирование корректности математических операций
- Проверка целостности данных после операций

8. ДЕМОНСТРАЦИЯ В РАБОТЕ

Программа демонстрирует:

- Выполнение всех определенных операций

- Корректность реализации через тесты
